## Reporting issues

<Note>
  Make sure you're reporting issues for the correct product. The community project is separate from commercial OpenVPN products.
</Note>

### OpenVPN open source

For bugs in the community OpenVPN project:

<Steps>
  <Step title="Search existing issues">
    Check if your issue is already reported at [GitHub Issues](https://github.com/OpenVPN/openvpn/issues)
  </Step>
  
  <Step title="Gather information">
    Collect:
    - OS and version
    - OpenVPN version
    - Configuration files (remove sensitive data)
    - Error messages and logs
    - Steps to reproduce
  </Step>
  
  <Step title="Create issue report">
    Open a new issue at [https://github.com/OpenVPN/openvpn/issues/new](https://github.com/OpenVPN/openvpn/issues/new)
  </Step>
  
  <Step title="Provide details">
    Include:
    - Clear description of the bug
    - Steps to reproduce
    - Expected vs actual behavior
    - Version information
    - Relevant configuration (sanitized)
  </Step>
</Steps>

## Before asking for help

To get the best help quickly:

### Check the documentation first

1. Read the relevant section in the [man page](http://openvpn.net/man.html)
2. Review the [HOWTO guide](http://openvpn.net/howto.html)
3. Search the [FAQ](https://community.openvpn.net/openvpn/wiki/FAQ)
4. Look for similar issues on [GitHub](https://github.com/OpenVPN/openvpn/issues)

### Prepare your question

Include:
- What you're trying to accomplish
- What you've already tried
- Specific error messages (exact text)
- Relevant configuration (with secrets removed)
- OS and OpenVPN version information
- Network topology (if relevant)

### Sanitize sensitive information

<Note>
  Always remove sensitive information before sharing:
  - Private keys and certificates
  - Passwords and authentication credentials
  - Real IP addresses (use examples like 192.0.2.1)
  - Domain names that reveal your infrastructure
</Note>
